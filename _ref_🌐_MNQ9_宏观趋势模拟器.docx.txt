🌐  MNQ9 宏观趋势模拟器 — 使用说明 v1.1（外发版·专业完整版） MNQ9 Simulator: Macro Dynamics & Confidence Kernel Model MNQ9 是一套基于“信心核模型（Confidence Kernel Model）”的宏观趋势模拟器，通过“事件波（Future Shocks）”与“宏观信心场（Macro Confidence Field）”的相互作用，生成市场的结构性趋势序列。 它适用于： 宏观经济分析 市场趋势判断 事件冲击模拟 多因子融合预测 金融与经济 AI 的输入层 MNQ9  结构清晰、参数简洁、稳定性高、易于扩展 ，适合科研人员、量化团队和行业研究者使用。 1. 模型整体结构（Model Overview） MNQ9 由三个核心模块组成： ① Ω（信心主核） 市场底层信心的惯性部分，负责趋势的“主方向”。 ② φ_future（未来事件波） 外部事件、数据公布、政策、新闻 NLP 所形成的“冲击向量”。 ③ B_conf（宏观信心场） 由宏观指标形成的背景信心强度，用于调节事件作用的有效性。 这三个模块共同形成趋势： 其中： λ 为信心衰减系数。 2. 输入数据要求（Data Requirements） MNQ9 可以在  日度 / 周度 / 月度  等不同频率运行，只需保证全部输入序列对齐。 2.1 事件数据（必需） 事件冲击序列 φ_series 必须是一个  数值时间序列 ： φ_series = [0.2, -0.3, 0.0, 0.1, ...] 来源可以包括： 来源类别 示例 NLP 新闻流 利好/利空事件自动提取 政策事件 降准、利率、财政刺激 经济数据公布 CPI、非农、PMI 企业数据 财报、盈利预告 全球事件 地缘政治、重大变动 建议范围： -0.5 ~ 0.5（可自动归一化） 2.2 宏观基础数据（可选但推荐） 以字典形式传入： B_conf = {     "M2": 0.18,     "PMI": 0.05,     "DR007": -0.10 } 所有字段建议归一化到 -1 ~ 1。 推荐使用的宏观字段： 字段 建议方向 说明 M2 正向 货币总量，越多越利好 PMI 正向 景气度，> 50 强、< 50 弱 DR007 负向 资金成本，越高越利空 社融 正向 信贷扩张程度 LPR 负向 利率环境 出口增速 正向 外需强弱 地产销售 正向 居民部门情绪代理 就业数据 正向 影响消费预期 你可以根据行业需求添加字段，只需归一化即可。 2.3 参数（可选） λ = 0.01 ~ 0.05  # 推荐范围 λ 越大：趋势越短、越快 λ 越小：趋势越长、越平滑 3. 模型接口（API Specification） MNQ9 提供简单直接的 API，适合集成到研究系统或 AI 系统中。 3.1 初始化 sim = MNQ9Simulator() 3.2 设置宏观信心场 sim.set_macro_confidence(B_conf) 3.3 设置事件冲击 sim.set_future_wave(φ_series) 3.4 运行模拟 Ω_series = sim.run_series(steps=len(φ_series)) 输出：趋势序列（list 或 numpy array） 例： [0.0, 0.08, -0.01, 0.20, -0.12, ...] 4. 模型细节（Model Mechanics） 4.1 信心主核（Ω）更新机制 Ω = Ω + φ_effective - λ * Ω 这意味着： 趋势受事件强化 趋势随时间缓慢衰减 形成自然的平滑走势 4.2 事件波（φ_future）作用方式 φ_effective = φ_future * tanh(kernel - 0.5) tanh 的作用： 限制极端值 使信号自然饱和 让事件的影响可控、平滑 4.3 宏观信心核（kernel）构建 kernel = w_M2*M2 + w_PMI*PMI + w_DR07*(-DR007) + ... 权重默认平均，也可以扩展为专业权重。 5. 调参指南（Tuning Guide） ✔  事件强度（φ） 小事件：±0.05 中事件：±0.2 大事件：±0.4（政策、地缘等） ✔  宏观信心字段（B_conf） 建议统一归一化方式，例如： 原值 归一化 PMI（50 = 0，对增量做比例） (PMI - 50) / 10 DR007（逆向） -(DR007 - 2.0) / 2.0 M2（同比） (M2% - 8%) / 8% ✔  λ（衰减参数） λ 适用周期 0.03–0.05 日度 0.015–0.03 周度 0.005–0.02 月度/季度 6. 示例（Complete Example） from mnq9_simulator import MNQ9Simulator φ_series = [0.2, -0.1, 0.0, 0.3, -0.4] B_conf = {     "M2": 0.25,      "PMI": 0.1,     "DR007": -0.2 } sim = MNQ9Simulator() sim.set_macro_confidence(B_conf) sim.set_future_wave(φ_series) Ω_series = sim.run_series(steps=len(φ_series)) print(Ω_series) 输出： [0.0, 0.07, -0.02, 0.20, -0.25] 意义： 第二期利空 → 下跌 第三期无事件 → 稳定 第四期大幅利好 → 强上升 第五期极端利空 → 剧烈回落 7. 应用场景（Use Cases） MNQ9 可用于： 🔹  宏观周期分析 🔹  趋势预测 🔹  多因子模型 🔹  事件冲击模拟 🔹  金融 AI 的输入层 🔹  研究报告自动生成 🔹  金融 NLP + 动力学大模型 🔹  CTA、趋势策略、择时策略 它特别适合做： “下一阶段趋势方向” “事件后影响持续多久” “宏观环境如何调节事件冲击” “不同市场的结构强弱对比” 8. 扩展方向（Future Extensions） 你未来可以基于 MNQ9 做： MNQ9-Pro（多核信心场） MNQ9-MultiShock（事件叠加模型） MNQ9-Factor（自动因子生成） MNQ9-LLM（金融大模型融合） MNQ9-Global（全球指数/大宗商品联动） MNQ9-AI（真正的金融智能体） 🧩  MNQ9 数据预处理标准 v1.0 MNQ9 是一个  时间序列模型 ，数据输入质量对最终趋势输出影响非常大。 因此，输入数据必须满足以下标准。 1. 数据结构要求（Data Structure） MNQ9 接受两类数据： （1）事件序列 φ_series（必需） 类型： 一维时间序列（list / array / pandas.Series） 格式：每个时间点一个事件强度 长度：任意，但通常 >= 20 期较为稳定 （2）宏观信心字段 B_conf（可选但推荐） 类型：字典（dict） 每个 key 一个宏观指标 每个 value 是 归一化后的数值（-1 ~ 1） 示例： φ = [0.2, -0.1, 0.0, 0.3] B_conf = {"M2": 0.18, "PMI": 0.05, "DR007": -0.12} 2. 数据清理（Data Cleaning） 2.1 缺失值 必需规则： 事件缺失 → 填 0 宏观指标缺失 → 前向填充（ffill） 全缺失序列禁止输入 2.2 异常峰值（Spikes） 超过 5 倍标准差 → clip 到 ±3σ 避免极端事件导致 φ_effective 爆炸 2.3 数据类型 要求： float 或 int 不接受字符串 不接受 None 3. 时间对齐（Time Alignment） MNQ9 必须保证 所有序列的时间轴一致 ： φ_series:       t0 t1 t2 t3 t4 ... M2 series:      t0 t1 t2 t3 t4 ... PMI series:     t0 t1 t2 t3 t4 ... DR007 series:   t0 t1 t2 t3 t4 ... 处理规则： 若频率不同（如 PMI 月度、DR007 日度），必须： 统一频率（建议使用“转换到事件频率”） 月度指标 → 日度使用当月值填充 周数据 → 日度线性插值 不允许时间错位 （例如事件在 t1，但宏观在 t2） 4. 数值归一化（Normalization） MNQ9 的所有宏观指标  必须归一化到 -1 ~ 1 。 这样输入才具有可比性，不会产生维度冲突。 推荐归一化规则如下： 4.1 正向指标（值大 → 趋势强） 如 M2、PMI、社融、工业增加值： 建议 k = 2 ~ 3。 4.2 逆向指标（值大 → 趋势弱） 如 DR007、LPR、利率： 4.3 百分比指标 如出口增速、CPI： （CPI 10% → 0.5） 4.4 PMI（特别指标） PMI 特别适合作为： PMI = 55 → +0.5 PMI = 45 → -0.5 非常直观。 5. 特征构建（Feature Engineering） 建议至少构建以下字段： ✔  5.1 宏观信心三角（推荐组合） M2_norm（流动性） PMI_norm（景气度） DR007_norm（资金成本） 这是最稳定的三元组，也是 MNQ9 的标准输入。 ✔  5.2 多周期平滑（可选） 对高噪声指标（如 DR007）可使用： 3-day EMA 7-day EMA 14-day EMA 例： DR007_norm = ema(DR007_norm_raw, span=7) 6. 事件序列预处理（φ_future） 事件强度 φ_future 是决定趋势转折的关键元素。 推荐使用以下规则（你可以做 NLP 自动识别）： 6.1 事件强度标度 分类 数值范围 示例 强利好 +0.30 ~ +0.50 政策刺激、降准、利率下调 中利好 +0.10 ~ +0.25 经济数据超预期 弱利好 +0.05 公司利好、行业消息 中性 0 无重要事件 弱利空 -0.05 行业小利空 中利空 -0.10 ~ -0.25 经济数据不及预期 强利空 -0.30 ~ -0.50 地缘冲击、金融风暴 6.2 多事件叠加 如果一天有多个事件： 然后再 clip 至区间： [-0.6 , +0.6] 确保稳定性。 6.3 事件衰减（可选） 重大事件可以有“延迟效应”： τ = 1~3（衰减天数） 7. 宏观信心场预处理（B_conf） 宏观指标必须先归一化，然后传入： B_conf = {     "M2": M2_norm,     "PMI": PMI_norm,     "DR007": DR007_norm } 如果没有某些字段，可以缺省。 权重默认平均： 也可用自定义权重： sim.weights = {"M2":0.5, "PMI":0.3, "DR007":0.2} 8. 参数标准化（Parameter Setting） ✔  λ：信心衰减 适用周期： λ 适用频率 0.03~0.05 日度 0.015~0.03 周度 0.005~0.02 月度 ✔  数据必须时间对齐 ✔  所有指标必须归一化 ✔  事件必须限定在合理区间 9. 最佳实践（Best Practices） 为了获得最佳效果，请遵循以下准则： 👍  9.1 宏观三角必用（M2 / PMI / DR007） 它们代表： 资金 景气 成本 是趋势的基础。 👍  9.2 使用 NLP 事件流增强 φ_future 这是 MNQ9 最强的部分。 👍  9.3 低噪声处理 所有指标最好做噪声过滤（EMA 或 Kalman Filter）。 👍  9.4 避免使用过多特征 3–7 个字段即可获得最佳相干性。 👍  9.5 数据频率越高，λ 越大 这是模型的内在规律。
