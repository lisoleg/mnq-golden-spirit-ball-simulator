## 1. 产品目标

### 现状问题
当前 MNQ 使用 Python tkinter GUI（`mnq_dashboard.py`），存在以下痛点：
- **本地受限**: 只能在安装 Python 的 Windows 机器上运行，无法远程访问
- **界面陈旧**: tkinter 界面粗糙，缺乏现代 Web 的交互体验
- **数据孤立**: 实验结果难以分享、展示和协作
- **可视化弱**: 缺乏实时动态图表和 3D 可视化能力

### 产品目标
将 MNQ 从桌面应用升级为 **Web 仪表盘**，实现：
1. **随时随地访问**: 通过浏览器远程使用，支持多设备
2. **现代交互体验**: 响应式设计，实时数据可视化
3. **结果可分享**: 生成分享链接，导出报告
4. **扩展性强**: 为后续 AI 集成、云部署预留接口

作为一名新用户，我希望通过交互式引导了解各面板功能，并一键运行示例实验，以便快速上手 MNQ 系统。

## 3. 需求池

### P0 (必须实现)
| 需求 | 描述 | 验收标准 |
|------|------|----------|
| **Flask REST API** | 封装 `mnq_core.py` 所有算法模块 | 提供 `/api/experiment/run`、`/api/kernel/status` 等接口 |
| **FrozenKernel 面板** | 显示冻结核状态、参数调节 | 可实时调整参数并查看结果 |
| **MASS_FACE 读数** | 显示面部识别算法读数 | 实时更新，支持历史查询 |
| **实验运行器** | 支持 18 种 CLI 实验的 Web 化 | 可在界面选择实验并查看输出 |
| **响应式布局** | 适配桌面和平板 | 在 1920x1080 和 1280x720 正常显示 |

### P1 (应实现)
| 需求 | 描述 |
|------|------|
| **SCF/CGD 面板** | 显示拓扑度、软约束等指标 |
| **MNQ9 置信度面板** | 显示宏观趋势模拟结果 |
| **MNQ-Deep Transformer** | 集成深度学习模块 |
| **κ-Snap 浏览器** | 查看和导出 κ-快照数据 |
| **实验历史** | 保存实验记录，支持对比 |

### P2 (可延后)
| 需求 | 描述 |
|------|------|
| **3D 可视化** | 金灵球网络拓扑 3D 渲染 |
| **实时协作** | 多用户同时查看和标注 |
| **报告生成** | 一键导出 PDF/PowerPoint 报告 |
| **移动端适配** | 支持手机浏览器访问 |
| **AI 辅助分析** | 自动解读实验结果 |

## 4. UI 设计稿

### 整体布局
```
+--------------------------------------------------+
|  [Logo] MNQ Web Dashboard    [用户] [设置]       |  ← 顶部导航栏
+--------+-----------------------------------------+
| 侧边栏  |  主内容区                               |
|        |                                         |
| 实验   |  [FrozenKernel 面板]                    |
|  Frozen|                                         |
|  MASS  |  +------------------+                  |
|  SCF/  |  | 冻结核状态图     |                  |
|  MNQ9  |  |                  |                  |
|  Deep  |  +------------------+                  |
|  κ-Snap|                                         |
| 历史   |  [参数调节区]                          |
|        |                                         |
+--------+-----------------------------------------+
```

### 关键组件
1. **侧边栏导航**: 图标 + 文字，折叠/展开
2. **FrozenKernel 面板**: 状态卡片 + 实时折线图 + 参数滑块
3. **MASS_FACE 读数**: 仪表盘 + 数字指标卡片
4. **实验运行器**: 实验列表 + 输出日志终端（仿终端样式）
5. **顶部状态栏**: 系统状态、用户信息、设置入口

### 设计风格
- **色彩**: 深色主题（科技感），主色 `#1890ff`（蓝色）
- **字体**: 等宽字体用于代码/日志，无衬线字体用于正文
- **图表**: 使用 Recharts 或 ECharts，支持暗色主题

## 5. 待确认问题

1. **部署方式**: 是本地部署（类似原 tkinter）还是支持云端部署？
2. **用户认证**: 是否需要登录系统？还是公开访问？
3. **数据持久化**: 实验历史存储在哪里（本地 SQLite 还是云端数据库）？
4. **实时性要求**: 仿真计算是否需要 WebSocket 推送进度，还是轮询即可？
5. **兼容性**: 是否需要支持 Python 3.8+ 以外的环境？
6. **性能要求**: 大规模仿真（如 1000+ 节点）时，前端是否需要分页或虚拟化？
